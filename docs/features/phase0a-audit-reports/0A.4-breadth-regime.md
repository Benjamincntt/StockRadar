# 0A.4 — Audit Market Breadth / Regime

## Tóm tắt 1 đoạn

**Có sẵn 1 phần.** Hệ thống đã có một dạng **"regime sơ khai" 3 mức** (`MarketWyckoffPhase = Favorable/Neutral/Unfavorable`) tính **thuần từ VNINDEX** (Trend + %change, hoặc %5d + MA20), lưu **gắn per-opportunity** theo ngày (cột `MarketPhase` trên `DailyOpportunities`/`EarlyRecoveryRadar`/...), và được tiêu thụ trong Buy Score, hit-probability, size overlay, VIP alert multiplier. **RS percentile toàn universe đã có** (`BuildRsPercentile`). Nhưng **TRẮNG** ở: (1) Market Breadth thực thụ (%>MA20/50, advance-decline, số mã sàn/trần, %new low, median toàn thị trường), (2) enum regime Panic/Stabilizing/ReboundConfirmed/Normal, (3) snapshot breadth/regime **cấp thị trường** độc lập trong DB, (4) API endpoint expose regime riêng, (5) hysteresis (nâng chậm/hạ nhanh). → Phase 0B phải **code mới gần như hoàn toàn** phần breadth + regime 4-state + hysteresis + snapshot + endpoint.

## Hiện trạng

### Breadth

- **Không.** Grep `MarketBreadth/Breadth/AdvanceDecline/PctAboveMA20/FloorCount/CeilingCount`: 0 kết quả trong code backend (chỉ ở docs Phase 0A).
- Metric gần nhất: **median %5d theo NGÀNH** (không phải toàn thị trường) trong `SmartMoneyOpportunitySelector.MedianChange5d` (`:254-268`) — chỉ để xếp hạng ngành, không đếm sàn/trần, không %>MA.
- **Lưu snapshot breadth: Không.**

### Regime

- **Có (sơ khai, 3 mức).** enum `MarketWyckoffPhase`; **không** có Panic/Stabilizing/Rebound.
- **Quy tắc phân loại**: có **HAI** bộ độc lập, đều **stateless**, đều chỉ dựa VNINDEX:
  - **(A) Real-time** — `ClassifyMarket(index)`: `Uptrend→Favorable`, `Sideway→Neutral`, ngược lại `ChangePercent < -1.5% → Unfavorable` else `Neutral`.
  - **(B) As-of lịch sử** — `ClassifyMarketPhase(indexHistory, asOfIdx)`: dùng `%5d + MA20` của VNINDEX (`change5d≥1 & aboveMA→Favorable`; `change5d≤-1.5 hoặc (<0 & !aboveMA)→Unfavorable`; else `Neutral`).
- **Hysteresis: Không** (tính lại mỗi phiên, không nhớ regime trước).
- **Snapshot theo ngày**: chỉ **gắn per-opportunity** (cột `MarketPhase`), **không** có bảng regime cấp thị trường riêng.

```csharp
// backend/StockRadar.Domain/Services/SmartMoneyOpportunitySelector.cs:148-156
private static MarketWyckoffPhase ClassifyMarket(MarketIndex index) =>
    index.Trend switch
    {
        MarketTrend.Uptrend => MarketWyckoffPhase.Favorable,
        MarketTrend.Sideway => MarketWyckoffPhase.Neutral,
        _ => index.ChangePercent < -1.5m
            ? MarketWyckoffPhase.Unfavorable
            : MarketWyckoffPhase.Neutral
    };   // ← chỉ VNINDEX trend + %chg, KHÔNG breadth
```

## Đề xuất

- [x] **Có một phần → Phase 0B bổ sung / code mới phần lớn.**
  - **Code mới**: `MarketBreadthAnalyzer` (tính 8+ metric từ toàn universe), `MarketRegimeClassifier` 4-state với **hysteresis** (2 phiên cải thiện mới nâng, 1 phiên xấu hạ), entity `MarketBreadthSnapshot` (bảng cấp thị trường theo `TradingDate` + `Version`), endpoint `GET /api/v1/reversal-bounce/market-regime`.
  - **Tái dùng**: `BuildRsPercentile` (RS improving), điểm chạm pipeline "quét toàn universe" trong `DailyAnalysisRunner` (chạy breadth **trước** analyzer), `SmartMoneyMarketContext` để lấy sẵn universe + index history.
  - **Lưu ý không đụng**: 3-mức `MarketWyckoffPhase` hiện tại đang được Buy Score/size/VIP alert dùng — **giữ nguyên**, regime 4-state của ReversalBounce là **hệ độc lập** (spec §4.4). Không map/ghi đè `MarketWyckoffPhase`.
