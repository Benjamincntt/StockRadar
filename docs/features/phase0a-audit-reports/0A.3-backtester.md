# 0A.3 — Audit Backtester Fill Logic

## Tóm tắt 1 đoạn

Backtester chính là `SmartMoneyBacktestRunner` (impl `IBacktestService`, endpoint `GET /api/v1/backtest/smartmoney`). Fill logic **cực kỳ đơn giản và lạc quan**: **entry = Close(T)** (đúng phiên ra tín hiệu, không có độ trễ mở cửa T+1), **exit = Close(T + holdSessions)** với `holdSessions` cố định (mặc định 5, clamp 2–20). **Không** có slippage/phí/thuế (return gross), **không** mô phỏng sàn/khóa sàn/không khớp lệnh, **không** tôn trọng ràng buộc T+3 (`MinTradingSessionsToSell` chỉ dùng cho Master Alert thật, backtest bỏ qua). Vòng lặp theo **phiên EOD**, không intraday. Không dùng High/Low T+1 để xác nhận tín hiệu trong replay chính (không look-ahead ở phần này). → ReversalBounce **không thể** tái dùng nguyên trạng: cần một module fill riêng (Open T+1, T+3 gate, floor-lock defer, slippage 2 tầng, gap-entry cancellation).

## Fill logic hiện tại

### Entry

- **FillPrice = Close(T)** — giá đóng cửa chính phiên ra tín hiệu. Không có slippage.
- Trích đoạn: `SmartMoneyBacktestRunner.cs:106-112`

```csharp
// backend/StockRadar.Infrastructure/MarketData/SmartMoneyBacktestRunner.cs:106-112
var stock = universe.First(s => s.Symbol.Equals(pick.Symbol, StringComparison.OrdinalIgnoreCase));
var entryPrice = stock.History[^1].Close;              // ← Close(T), cùng phiên tín hiệu
var exitPrice = TradingSessionMath.GetForwardPriceAtSessions(
    FindFullHistory(all, pick.Symbol),
    asOfDate,
    holdSessions);
```

### Exit

- **FillPrice = Close(T + holdSessions)** — hold cố định rồi thoát tại close. Không stop-loss / take-profit / trailing.
- Trích đoạn: `TradingSessionMath.cs:105-109` + `71-80`

```csharp
// backend/StockRadar.Domain/Services/TradingSessionMath.cs:105-109
public static decimal? GetForwardPriceAtSessions(
    IReadOnlyList<OhlcvBar> history,
    DateOnly entryDate,
    int sessions) =>
    GetCloseOnOrAfter(history, AddTradingSessions(entryDate, sessions));
```

```csharp
// backend/StockRadar.Domain/Services/TradingSessionMath.cs:71-80
public static decimal? GetCloseOnOrAfter(IReadOnlyList<OhlcvBar> history, DateOnly date)
{
    foreach (var bar in history.OrderBy(b => b.Date))
    {
        if (bar.Date >= date)
            return bar.Close;          // ← luôn lấy được Close, không mô phỏng "không bán được"
    }
    return null;
}
```

### T+3 / MinTradingSessionsToSell

- **Hardcode/config ở**: `MasterAlertOptions.cs:49` (default 3), `appsettings.json:194`.
- **Có config**: có (`MinTradingSessionsToSell`), nhưng **backtester KHÔNG tham chiếu**. Nơi dùng thật: `TopOpportunityVipAlertEvaluator.cs:157` (`canSell = sessions >= cfg.MinTradingSessionsToSell`).
- Backtest dùng `holdSessions` cố định (clamp 2–20) thay cho quy tắc T+3 → có thể "bán" tại T+2, vi phạm T+2.5.

### Slippage

- **KHÔNG TÌM THẤY.** Grep `Slippage|Fee|Commission|thuế|phí` trong runner: 0 match. Return tính gross `(exit-entry)/entry*100`.
- (Ghi chú: `SuccessThresholdPercent`/`FlatMinPercent` trong `ClassifyOutcome` chỉ là ngưỡng phân loại Good/Flat/Bad, không phải chi phí.)

### Floor-lock / không bán được

- **KHÔNG TÌM THẤY.** Không xử lý trần/sàn/dư bán. Nếu thiếu bar đích → `if (exitPrice is null or <= 0) continue;` (bỏ lệnh, coi như không tồn tại — không phải mô phỏng khóa sàn).

## Đối chiếu yêu cầu ReversalBounce

| Yêu cầu trong reversal-bounce.md | Hiện trạng | Đạt? |
|---|---|---|
| Fill tại Open(T+1), không Close(T) | Entry = **Close(T)** | ❌ |
| Hard exit chỉ từ T+3 | Dùng `holdSessions` cố định, bỏ qua `MinTradingSessionsToSell` | ❌ |
| Floor-lock defer exit | Không mô phỏng sàn; luôn fill Close | ❌ |
| Slippage 2 tầng (base + gap) | Không có slippage | ❌ |
| Gap-entry cancellation | Không có (entry = Close T, không xét gap Open T+1) | ❌ |
| Không dùng High/Low T+1 để xác nhận tín hiệu | Replay chính chỉ dùng Close → **không** look-ahead High/Low | ✅ |
| Time-step theo phiên EOD | Có (`TradingDaysBetween`, `SmartMoneyBacktestRunner.cs:77`) | ✅ |

Tham chiếu thêm: vòng lặp EOD `SmartMoneyBacktestRunner.cs:77-88`; `ComputeSwingPath` (MFE/MAE dùng High/Low) tồn tại tại `TradingSessionMath.cs:118-165` nhưng **không** được `RunSmartMoneyAsync` gọi.
